Flutter listview学习

发表于 2020-06-03 | 分类于 Flutter
默认参数

  • itemExtent：该参数如果不为null，则会强制children的“长度”为itemExtent的值；这里的“长度”是指滚动方向上子组件的长度，也就是说如果滚动方向是垂直方向，则itemExtent代表子组件的高度；如果滚动方向为水平方向，则itemExtent就代表子组件的宽度。在ListView中，指定itemExtent比让子组件自己决定自身长度会更高效，这是因为指定itemExtent后，滚动系统可以提前知道列表的长度，而无需每次构建子组件时都去再计算一下，尤其是在滚动位置频繁变化时（滚动系统需要频繁去计算列表高度）。

  • shrinkWrap：该属性表示是否根据子组件的总长度来设置ListView的长度，默认值为false 。默认情况下，ListView的会在滚动方向尽可能多的占用空间。当ListView在一个无边界(滚动方向上)的容器中时，shrinkWrap必须为true。

    child 高度会适配 item填充的内容的高度,我们非常的不希望child的高度固定，因为这样的话，如果里面的内容超出就会造成布局的溢出。

  • addAutomaticKeepAlives：该属性表示是否将列表项（子组件）包裹在AutomaticKeepAlive 组件中；典型地，在一个懒加载列表中，如果将列表项包裹在AutomaticKeepAlive中，在该列表项滑出视口时它也不会被GC（垃圾回收），它会使用KeepAliveNotification来保存其状态。如果列表项自己维护其KeepAlive状态，那么此参数必须置为false。

    在 一个 Scaffold + Appbar + Tabbar + PageView 的组合页面中TabBarView切换key改变，子控件的initState 和 build 都会执行。

    如果该页面为listview构成，使器addAutomaticKeepAlives为flase 则initState 不会被二次创建。

    或者每个 Tab 对应的 StatefulWidget 的 State ，需要通过with AutomaticKeepAliveClientMixin ，然后重写 @override bool get wantKeepAlive => true; ，就可以实不重新构建的效果了

  • addRepaintBoundaries：该属性表示是否将列表项（子组件）包裹在RepaintBoundary组件中。当可滚动组件滚动时，将列表项包裹在RepaintBoundary中可以避免列表项重绘，但是当列表项重绘的开销非常小（如一个颜色块，或者一个较短的文本）时，不添加RepaintBoundary反而会更高效。和addAutomaticKeepAlive一样，如果列表项自己维护其KeepAlive状态，那么此参数必须置为false。

默认构造函数

通过此方式创建的ListView和使用SingleChildScrollView+Column的方式没有本质的区别。

只适用于子组件较少的情况

ListView.builder
  • itemBuilder：它是列表项的构建器，类型为IndexedWidgetBuilder，返回值为一个widget。当列表滚动到具体的index位置时，会调用该构建器构建列表项。
  • itemCount：列表项的数量，如果为null，则为无限列表。
ListView.separated

itemCount：必须为固定值>0 且 不能为null。

比builder 多了一个separatorBuilder，提供分割构造器

甚至还可以混入广告，不需要对列表项中间的主列表进行任何修改。

ListView.custom

需要重写SliverChildBuilderDelegate

实现监听其中didFinishLayout方法，可以监听可见列表中，第一个位置和最后一个位置的index

实现其中findChildIndexCallback，若其itemview 为stateful widget，混入AutomaticKeepAliveClientMixin 并重写wantKeepAlive 为true，创建时设置key，在修改item状态后，改变items值能保存其状态。

gridview

gridview 构造参数大部分与listview相同，但是控制widget layout的方法不同，为gridDelegate。类型是SliverGridDelegate，它的作用是控制GridView子组件如何排列(layout)。

crossAxisCount与 childAspectRatio决定子元素大小。

gridview 默认子元素大小都为一致，默认实现不了
