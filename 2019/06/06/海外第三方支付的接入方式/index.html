海外第三方支付的接入方式

发表于 2019-06-06 | 分类于 android海外调研
Paypal
1.账号配置
  • 前往https://www.paypal.com/注册商家账号
  • 前往https://developer.paypal.com/，使用刚注册的商家账号登录
  • 点击右上角名字，选择Dashboard
  • 在左边的导航栏中点击 Sandbox 下的 Accounts
  • 进入Acccouts界面后，可以看到系统有两个已经生成好的测试账号，但是我们不要用系统给的测试账号，自己创建两个，一个是商家账号，一个是个人账号
  • 创建应用，生成用于测试的clientID 和 密钥
2.代码接入

  • 官方android sdk https://github.com/paypal/PayPal-Android-SDK

  • 在app级的gradle中引入服务

  • 在onCreate中启动支付服务

  • 在onDestory中停止服务

  • 配置信息

    CONFIG_ENVIRONMENT 调试环境

  • 生成支付实体，在需要的地方调起支付

  • 在类的onActivityResult 里进行回调结果的处理

Stripe
1.账号配置
  • 前往https://stripe.com/注册商家账号
  • 在https://dashboard.stripe.com/test/dashboard激活账号并获取密钥
  • 2.方案实现

  • 方案一：

  1. 收集卡信息，生成token

  2. 将token发送给服务端，由服务端发起购买请求

  3. 服务端返回支付成功or失败

    缺陷：缺少3D验证，一些发卡机构需要3D验证才能扣款成功，否则会被拒
